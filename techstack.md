Technical Specification: Pokedex AI Stack1. System ArchitectureThis project follows a decoupled Frontend/Backend architecture. The Frontend handles the heavy UI/UX and camera stream, while the Backend handles the AI inference.2. Frontend (React + Vite)State Management: React useState for handling the isScanning, resultData, and animationComplete states.Camera Integration: Use navigator.mediaDevices.getUserMedia with facingMode: "environment" to force the rear camera.Styling: Tailwind CSS for layout, with a custom pokedex.css for the "plastic" textures and button shadows.Animations: framer-motion for the sliding "Dex" entries and the initial Pokéball split.3. Backend (FastAPI + Python)Model: ResNet50 (Residual Network with 50 layers).Reasoning: Highly efficient for mobile-web latency; pre-trained on ImageNet which contains hundreds of animal sub-classes.Preprocessing: Images must be resized to $224 \times 224$ pixels and normalized using the ImageNet mean and standard deviation:$$\mu = [0.485, 0.456, 0.406], \sigma = [0.229, 0.224, 0.225]$$API Endpoint: POST /v1/scanInput: Multi-part Form Data (Image file).Output: JSON object containing label, confidence, and dex_entry.4. AI & Data LogicTo make the descriptions sound like a real Pokedex, we will use a Mapping Layer.Logic: The ResNet50 output (e.g., "Golden Retriever") is used as a key to look up a custom "Lore" description in a local species_data.json file.Fallback: If a specific lore entry doesn't exist, the system will use a generic template: "A member of the [Class] family often found in various habitats."
